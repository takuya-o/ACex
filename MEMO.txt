-masterブランチの作り方
masterブランチは孤立したブランチになってしまうが、開発ブランチの細かい
修正を見せない。

# merge --sqash でcommitが一つにまとまる!

$ git checkout -b releaseV1.0.0.0 master #安全のためにreleaseブランチで
$ git merge --Xtheirs --squash develop
$ git commit -m "1.0.0.0"

$ git commit --amend -C HEAD --date=2014-03-18T00:00:00
$ git rebase --committer-date-is-author-date HEAD~1

$ git co master        #よさげならmasterに戻って
$ git merge release    #マージ

$ git push             #upstreamリポジトリにpush
$ git push origin master     

$ git br -d release    #releasseブランチ削除
$ git co develop       #開発ブランチに戻って
$ git tag V0.0.0.3     #どこまでmasterにマージしたか忘れないようにタグ打ち
